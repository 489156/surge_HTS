# Step 1 — 데이터 소스 확정 (무료 MVP 우선)

목표: **전일 종가 대비 +100% 급등주의 *전일* 상태를 매일 박제**한다. 핵심은 정확도보다
"사후 복원이 불가능한 값(즉시포착)을 오늘부터 누적"하는 것.

## 시세(quote) 이중화 — SOXL/SOXS 추적 (2026-06-11)

`sources/quotes.py`의 3중 failover 체인. 모든 소비자(trading·duel·dashboard)가
`brokers.default_last_price`(60초 캐시)를 통해 이 체인을 탄다. `surge quotes --health`.

| 순위 | 공급자 | 키 | 성격 | 검증 |
|---|---|---|---|---|
| 1 | yfinance | 불필요 | 주 소스 | ✅ 라이브 |
| 2 | Finnhub REST | `SURGE_FINNHUB_API_KEY`(무료 60콜/분) | **진짜 벤더 이중화** | 키 등록 시 활성 |
| 3 | Yahoo chart API 직접(httpx) | 불필요 | 클라이언트 경로 이중화(yfinance 라이브러리 파손 대비; 인프라는 동일 Yahoo) | ✅ 라이브 |
| — | Stooq | 불필요 | **평가 후 탈락** — 이 망에서 전 엔드포인트 404 | ❌ |

SOXL/SOXS는 Prometheus `surge_quote{symbol=…}` 게이지로도 노출(Grafana 차트 가능).
MCP 레지스트리에는 주식/금융 커넥터 없음(검색 0건).

## MVP 결정

- **1차 소스 = yfinance** (API 키 불필요 → *오늘 바로 실행*). float/short/options/실적일을 한 번에 커버.
- **2차 소스(선택) = Finnhub** 무료 60콜/분. `FINNHUB_API_KEY` 설정 시에만 활성.
- **유료 자리만 확보**: 차입비용(borrow), Ortex 정밀 공매도, 소셜 API — 스키마에 컬럼은 두되 NULL 허용.

## 즉시포착(매일 안 모으면 영원히 손실) — 최우선

1. 옵션 IV·콜/풋 거래량 (`.option_chain`)
2. float·기관보유·공매도(지연이라도) (`.info`)
3. 애프터/프리마켓 갭은 일별 OHLCV의 다음날 시가로 근사
4. (후속) 차입비용·소셜 가속

> 차입비용/소셜은 무료로 과거 복원이 사실상 불가하므로, 무료 확보 경로가 생기는 즉시
> 박제 대상에 추가한다. 그전까지는 컬럼을 NULL로 유지(스키마 호환).

## 신규 상품 후보 조사 프로토콜 (2026-07-03 제정 — SK하이닉스 오판 사후조치)

**사고**: "SK하이닉스 레버리지가 존재하는가?"를 조사하며 "US listed"로 스코프를 좁힌
영어 검색만 수행 → 미국 나스닥 ADR 기반 신상품(7/13 상장 예정)만 발견하고
**"아직 상장 전 — 물리적으로 불가능"**이라고 오판. 실제로는 KRX에 KODEX·TIGER·
ACE·RISE·SOL 5개 운용사의 SK하이닉스 단일종목 레버리지(현물형, 2026-05-27 상장)가
**이미 두 달 전부터 거래 중**이었다. 원인은 4갈래:

1. **검색 스코프 자기충족적 오류**: 종목의 "주 상장 시장"을 확인하지 않고 질문에
   맞춰 검색어를 좁힘("US listed"로 검색하면 US 상품만 나오는 게 당연). 회사의
   본토 시장(SK하이닉스=KRX 000660)을 먼저 특정했어야 했다.
2. **자체 툴박스 미확인**: `000660.KS`는 이미 `duel/data.py`(아시아 선행 신호)와
   `rotation/chains.py`(KR 로테이션)에 등록되어 있었다 — 즉 "이 프로젝트가 이미
   이 종목의 KR 시장 접근권을 갖고 있다"는 증거가 코드베이스 안에 있었는데도
   교차 확인하지 않았다.
3. **최신성 편향**: 나스닥 ADR 상장(7/10)이 "First Mover"·"처음" 프레이밍의
   보도자료로 검색 결과를 지배해, "최근 뉴스가 많다 = 최초다"로 오인.
4. **단일 소스·단일 언어 검증**: 부정 판정("없다"/"불가능")을 영어 검색 1회로
   확정. 한국어 검색 또는 KRX/네이버금융 등 현지 소스로 교차검증하지 않았다.

**방지책 (차기 신규 상품 조사부터 의무 적용)**:

- [ ] 회사의 **주 상장시장**을 먼저 특정한다(원주가 KRX/TSE/기타면 그 시장부터
      검색; "US listed"처럼 시장을 검색어에 미리 못박지 않는다).
- [ ] `grep -rn "<티커/종목명>" src/` 로 **이 코드베이스가 이미 그 종목에 접근권을
      갖고 있는지** 먼저 확인한다 — 이미 등록된 종목이면 새 시장 리서치가 아니라
      기존 소스 확장 문제일 가능성이 높다.
- [ ] 부정 판정("상품이 없다"/"불가능하다")은 **서로 다른 언어·시장의 소스 ≥2개**로
      교차검증 전에는 확정하지 않는다. 1개 소스·1개 언어의 결과는 "미확인"으로
      보고하되 "없음"으로 단정하지 않는다.
- [ ] "최근 뉴스가 많다"를 "최초/유일하다"의 근거로 쓰지 않는다 — 보도량은
      최신성의 지표이지 존재 여부의 지표가 아니다.
