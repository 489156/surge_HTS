## 1. 접속 방법

1. PowerShell을 연다.
2. 프로젝트로 이동: `cd "C:\Users\khy48\Downloads\vibe code\STOCK"`
3. 실행: `uv run surge dashboard` → `Uvicorn running on http://127.0.0.1:8000` 확인.
4. 브라우저에서 **http://127.0.0.1:8000** 열기. **종료: 터미널에서 `Ctrl+C`.**
- 포트 충돌 시 `--port 8001`. 내 PC 전용(127.0.0.1)이라 외부 노출 없음.
- 화면은 6초마다 자동 새로고침. 서버가 켜진 동안에만 접속된다.
- `uv`가 멈추면(이 환경 이슈) 직접 실행: `.venv\Scripts\surge.exe dashboard` 또는 `.venv\Scripts\python.exe -m uvicorn surge.dashboard.api:app --port 8000`.

### 1.1 휴대폰(모바일)에서 보기 — 같은 Wi-Fi
1. PC와 휴대폰을 **같은 공유기/Wi-Fi**에 연결.
2. PC에서 **LAN 바인딩**으로 실행: `uv run surge dashboard --host 0.0.0.0`
   → 콘솔에 **`📱 휴대폰(같은 Wi-Fi) → http://192.168.x.x:8000`** 가 자동 표시됨(그 주소를 폰 브라우저에 입력).
3. 처음 막히면 **Windows 방화벽**에서 해당 포트(8000) 인바운드 허용.
- 레이아웃은 모바일 반응형(표는 가로 스크롤, 카드 1열). 세로 화면 그대로 사용 가능.
- **보안**: 조회(GET)는 LAN에서 자유. **제어(Run cycle·Kill)는 원격 차단이 기본값**(안전). 폰에서 제어까지 쓰려면 `.env`에 `SURGE_DASHBOARD_TOKEN` 설정 후 토큰 전송 필요. 신뢰된 가정용 Wi-Fi에서만 `--host 0.0.0.0` 사용.

## 2. 상단 바

| 요소 | 의미 |
|---|---|
| `mode` 알약 | 현재 모드 — **paper**(모의·기본) / live(§5 참조) |
| `kill switch` 알약 | **armed**(정상 대기) / **ACTIVE**(비상 정지 발동됨, 빨강) |
| 시계 | 마지막 새로고침 시각 |
| **▶ Run cycle** | 모의 트레이딩 **1회 실행** — 후보를 에이전트가 평가→토론→리스크 점검→(페이퍼)체결. 실제 주문 아님 |
| **■ Kill switch** | **비상 정지** — 미체결 주문 취소 + 포지션 청산(페이퍼) + 신규 진입 차단 |
| **↺ Reset** | 비상 정지 **해제** — 다시 거래 가능 상태로 |

## 3. (A) 예측·검증 참조 계층 — 매일 보는 정보

### ① 검증 게이트 (가장 먼저 본다)
**"이 시스템을 믿고 돈을 걸어도 되는가?"** 에 한 줄로 답한다. ②③가 만든 예측을 각
전략의 **순진한 기준선**과 비교해, 운으로 설명되지 않는 초과가 있는지 통계로 판정한다.

| 컬럼 | 뜻 |
|---|---|
| 판정 | 🟢 검증된 엣지(z≥1.64·기준선+champion 동시 초과) · 🟡 미검증 · 🔴 표본부족(n<30) · ⛔ 기준선보다 유의하게 나쁨 |
| 전략 적중 / 기준선 / 초과 | 전략의 성적 / 비교 대상(찍기) / 그 차이(%p) |
| 표본 n | 전진 채점된 표본 수 |
| **엣지 막대** | 적중(채움)과 기준선(노란선)의 거리 — 한눈에 초과/미달 |
| **신호증거 미터** | **anytime-valid e-값** 진척(0→100%, e=1→20). 매일 봐도 거짓양성이 안 느는 peek-safe 증거 — 100%면 ⭐ 통계요건 충족. **미터=이항 e-값 하나**(보정된 단일 게이트). 미터 아래 **연속·페어**(수익크기·동일세션 대조)는 **보조진단**일 뿐 게이트 아님 — 트랙을 게이트에 더하면 FWER상 이항 검정력만 깎이므로(공짜 없음) 단일 검정을 유지 |

> **🟢가 하나도 없으면 ②③의 모든 출력은 '신호'가 아니라 '가설'이다.** 베팅 근거가 아니다.
> 🟢가 떠도 곧장 '신호'는 아니다 — ⭐ 격상 5조건(표본 n≥30·기간분산 4주+·**증거 e≥20**·
> 전·후반 유지·비용차감>0)은 [OPERATIONS.md] 참조. **e-값으로 신호 검증을 정직하게 앞당긴다**
> (엿보기 편향 없는 peek-safe 검정).

### ② 오늘밤 duel 콜 (US 레버리지)
오늘밤 미국장 **6개** 레버리지 페어(SOXL/SOXS·TQQQ/SQQQ·TECL/TECS·LABU/LABD·TNA/TZA·FAS/FAZ) **방향 예측**(읽기전용).
- `side` = 방향(강세 레그 티커) · `score` 양수=강세 · **`관망`도 정상 출력**(억지 베팅 안 함).
- **`현재` 칼럼 = Finnhub 실시간 시세**(동시 조회·60초 캐시) · 상단 줄 = **콜 생성시각 + 경과(age)**, 30h 초과 시 `⚠ 오래됨`.
- `확신` = 관망/하프/풀(●●○) — score 크기에 따른 베팅 비중(추천 아님).
- 진입/손절/익절 = ATR 기반 기계적 브래킷(추천 아님).
- **가격 게이지**(현재 칸 아래) = 손절(빨강 좌)↔익절(초록 우) 트랙 위 **파란 점=실시간 현재가**, 점선=진입. 24h 정적 콜이 실시간 어디에 있는지 한눈에.
- 하단 누적 = 지금까지 적중률(·**표본부족 n<30** 표시 시 결론 금지).
- **↑ ①의 'duel' 행이 이 콜들의 누적 성적을 판정한다 — 🟢 전엔 참고만.**

### ③ rotation 후보 (KR 회전)
한국 가치사슬 **회전 후보**(반도체·로봇 등 테마의 후방 노드). **KR 마감 후 1회/일(월–금)** 갱신.
- `✓` = 5조건 AND 게이트 통과(1순위 관찰) · `·` = 부분충족(연구용).
- `노드` = 가치사슬 위치 · `후방` = 핫노드에서 몇 단계 뒤(0=이미 달궈진 핫노드).
- **`분석가` 칼럼 = 분석일 종가**(저장값) · 상단 줄 = **분석 세션일 + 경과(age)**, 40h 초과 시 `⚠ 신규 분석 없음`.
- **행 클릭 → `현재가`(최근 종가·EOD)** + 체인 선두 가격연동(상관) + ATR 진입가 + 🏛 정책태그. (KR은 키리스 실시간 시세가 없어 현재가=최근 종가)
- **이미 급등한 핫노드(후방=0) 추격 금지.**
- **↑ ①의 'rotation' 행이 이들의 T+5(+10% 도달) 성적을 판정한다.**

### ④ 섀도 팩터 — "어떤 팩터를 추가할까" (진화 학습 루프)
라이브 투표에 **아직 없는** 후보 신호를 전진 채점한 표. 시스템이 *스스로* "이 신호를
고려했어야 했나"에 데이터로 답하는 곳.
- `적중률` = 그 팩터 단독의 전진 부호 적중 · `기준선 대비` = always-bull 대비 초과(%p).
- **양수(초록) + 유의**해야 추가 후보. 현재는 어떤 후보도 기준선 미초과 → 추가 없음(정직).
- 채움: `surge factors --backfill`(2년 451세션 즉시). 이후 매일 자동 누적.
- 핵심: **노이즈 팩터는 Šidák 보정 게이트가 자동 차단** — 과적합으로 진화하지 않는다.

## 4. (B) 모의(페이퍼) 트레이딩 엔진 — 별도 실험 계층

> ①②③ "예측"을 **자동 매매로 시험**하는 계층. **실제 자금 아님.** 아래 보유종목은
> 추천이 아니다(§4.1 기준 참조).

| 영역 | 보여주는 것 |
|---|---|
| **Account · 모의 계좌** | 평가액·현금·일/주/월 손익(손실한도 게이지)·주식비중 |
| **Equity curve · 자산 곡선** | 시작자본 대비 평가액 추이(최근 300포인트) + 누적수익률 — **소표본 주의** |
| **Open positions** | 현재 보유 모의 포지션 + 미실현손익(uPnL) + 포지션별 `수익%` · 하단 **코호트 승률/중앙값**(승자만 보지 말 것 — 생존편향 경고) |
| **Watchlist · surge 후보** | surge 점화 스크리너 상위(저유동·고RVOL) = 엔진의 **매수 유니버스** |
| **Recent decisions** | 엔진의 매매 결정 로그(행 클릭→에이전트 의견). net=종합점수, conf=확신도 |
| **Agent opinions** | 선택한 결정의 에이전트별 점수·추천·근거 |
| **Trade history** | 체결된 모의 거래 내역 |
| **Risk limits & state** | 자동 정지 한도(일/주/월 손실·최대 포지션·동시보유) — 넘으면 Kill switch 자동 발동 |
| **Pending live approvals** | 라이브용 사람-승인 대기열(§5 — 현재 실거래 미연결) |
| **Audit log / alerts** | 모든 행동의 감사 기록(킬스위치·체결·승인·정지) |

### 4.1 모의 계좌의 보유 기준 (중요)
보유 종목은 다음 파이프라인이 정한다:
**surge 후보 유니버스(상위 N)** → 에이전트 평가 → **토론 net≥55 매수(≥65 풀사이즈)** →
**리스크 엔진 사이징/거부** → (페이퍼)체결.

- 즉 매수 후보는 **surge 스크리너(미국 저유동 급등주)** 이며, **KR rotation과는 다른 thesis**다.
  "rotation의 나스닥판"이 **아니다** — 미국 쪽엔 가치사슬 회전 모듈이 아직 없다.
- 그리고 ①에서 **surge는 현재 ⛔(익일 상승률 45% < 동전 50%)** 다. 따라서 **이 모의 포지션은
  따라 살 대상이 아니라, '검증 안 된 신호로 매매하면 어떻게 되는가'를 측정하는 실험**이다.

## 5. 라이브(실거래) 모드의 실제 상태 — 반드시 알 것

**라이브 모드는 코드에 존재하지만, 의도적으로 작동하지 않는다(inert):**
1. 기본값은 `trading_mode=paper`. 라이브로 바꿔도 —
2. 라이브 브로커(Alpaca)의 `place_order`는 **자동 제출을 코드로 거부**(LiveBrokerGateError).
3. 실제 전송 경로 `submit_approved()`는 **코드 어디에서도 호출되지 않는다.**
→ 결론: **승인 버튼(✓)을 눌러도 실제 주문은 브로커로 전송되지 않는다.** DB 상태만 'submitted'로
바뀐다. **이 플랫폼은 사실상 페이퍼 전용**이며, "Pending live approvals"는 **미래 실거래를 위한
안전장치(HITL) 골격**이지 작동하는 실거래 기능이 아니다. 실거래를 켜려면 별도 구현 + 사람 승인 +
명시적 동의가 필요하다(§6 HITL — 비자동·비가역 행동은 항상 사람 승인).

## 6. 색·마크 범례
- 초록=양(+)/통과 · 빨강=음(−)/위험 · 회색=중립/없음.
- 🟢 검증 엣지 · 🟡 미검증 · 🔴 표본부족 · ⛔ 기준선 이하 · ✓ 게이트 통과 · 👑 active 변형 · 🔬 자동발굴 가설.

## 7. 안전·주의
- **🟢 전까지 ②③·모의 포지션은 전부 '가설'.** 일일 결과로 판단하지 말고 월간 누적·기준선 초과로.
- Kill switch/Reset/Run cycle/승인은 **모의 엔진 제어**다(실거래 아님).
- 외부 배포(0.0.0.0) 시에만 `SURGE_DASHBOARD_TOKEN` 필요 — 로컬은 불필요.
- CLI 전용 상세: `surge duel-gap`(갭원인) · `surge variants`(변형 리더보드) · `surge watch`(3-호라이즌).
