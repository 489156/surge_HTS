# Duel — 매일 밤 레버리지/인버스 페어 판정 엔진

## 빠른 확신 검증 (2026-07-03) — 페어별 수년 → 패밀리 단위 수개월

문제: 정직한 검증(anytime-valid e-value)은 페어마다 **플랫 프라이어에서 하루
1관측·얇은 엣지(≈53% vs ≈50%)**로 유의성을 재유도하므로 e≥20 도달에 수백~수천
세션(수년)이 걸린다. 신규 페어(NVDL/NVD)는 그 시계를 0에서 다시 시작한다.
`duel/verify.py`가 **개별적으로 건전한 3가지 가속기**를 결합해 이를 단축한다:

1. **교차 페어 풀링** (최대 지렛대): 적응 엔진은 *하나의* 메커니즘이다. "엔진이
   각 페어의 baseline을 이긴다"는 **공유 가설**을 전 페어 세션을 날짜순으로
   하나의 e-process에 엮어 검증 — 하루 ~7관측. 패밀리를 ~7배 빨리 검증하고,
   신규 페어는 "엔진이 작동함"을 즉시 상속받아 *자신이 이상치가 아님*만 보이면
   된다.
2. **프라이어 워밍**: plug-in 검정 마팅게일의 베팅 rate 추정을 리플레이 적중률로
   워밍(의사관측 `verify_prior_weight`≈15). **증거가 아니라 베팅 크기만** 조정
   — e는 여전히 1.0에서 시작해 전진 데이터로만 자란다(anytime-validity 불변).
   플랫 시작의 ~50스텝 워밍업 비용을 회수.
3. **McNemar 페어드**: 같은 세션에서 head-to-head 비교로 baseline 추정 슬랙을
   제거하고 불일치(discordant) 세션에 정보를 집중.

### 검증된 효과 (합성 대조, 300 시드) — 방법의 건전성을 분리 증명

| 시나리오 | 페어별(플랫) 중앙 세션 | 풀링(캘린더) 세션 | 위양성 |
|---|---|---|---|
| 실 엣지 0.55 | **764** (~3년) | **106** (~5개월) — **7.2배 단축** | — |
| 널 0.50 | — | **교차 안 함** | **2.3%** (≤α) |

- **환원 속성**: `prior_weight=0`이면 기존 `verdict.evalue_bernoulli`와 **정확히
  동일** — 워밍은 순수 가속이며 검정을 바꾸지 않는다(테스트로 못박음).
- **정직성 핵심**: 널(진짜 엣지 0)에서 풀링 e-value가 교차하는 비율 **2.3%**
  (α=0.05 이하). 즉 이 단축은 **통계적 검정력**이지 optional-stopping 위양성
  주입이 아니다. 풀링은 7개 개별검정을 1개 anytime-valid 프로세스로 합쳐
  다중비교 위양성도 오히려 줄인다.
- **실 아카이브 정직한 현실**: 실제 페어 엣지는 매우 얇다(traded 세션 vs 코인
  0.50~0.54). 방법은 SOXL/SOXS(후반부 0.497, 코인 이하)를 **올바르게 검증하지
  않는다** — 가짜로 통과시키지 않는다. 최종 심판은 언제나 라이브 전진 기록.

사용: `surge verify` (패밀리 e-value + 페어별 워밍/기존 e-value + 투영 세션 수).
매일 밤 `surge daily`의 learning_log에 `verify` 요약 박제. 프로덕션 전환 게이트가
아니라 **현황 표시** — 실제 자금 판단은 여전히 사람 게이트.

## 대시보드 적중/빗나감 이유 분석 (2026-07-21)

상시 대시보드는 "직전 세션 콜+결과"에 **왜 맞았나/틀렸나** 귀속 분석을 함께
표시한다(`export._attribution`). 박제된 컴포넌트 각각의 방향 투표를 실제
기초지수 시가→종가와 대조해: 맞은 신호·틀린 신호를 분리하고, 지배 신호가
방향을 잡았는지 한 줄 판정을 낸다. 누수 없음(박제 컴포넌트 + 실현 라벨만 사용).

첫 실측(2026-07-17, 적중 1/7): "빗나감 — 아시아 선행 등 신호는 하락 예측했으나
기초가 상승 반전(+2.56%)". 전 데스크가 하락을 가리켰는데 시장이 광범위 반등한
날 — 아시아 선행이 미국 반전을 놓친 것이 7페어 대부분의 공통 원인으로 드러났다
(금융 FAZ만 −0.30%로 적중). 사각지대 루프가 이런 신호충돌·전면오판 패턴을
변인화하는 것과 정확히 연결되는 진단 계층이다.

## 변동성 리스크 계층 + 필수매수 제약 (2026-07-20)

라이브 변동성 검증에서 두 가지가 드러났다: (1) 킬스위치는 지수 VIX만 보는데,
2026-07 SOXX 실현 변동성이 연율 68%까지 치솟는 동안 VIX는 ~16로 잠잠했다 —
지수는 평온한데 3배 상품이 하루 6~11% 출렁였다. (2) 사용자 정책: 매일 밤 최소
한 종목은 반드시 매수한다.

**실현 변동성 사이징 감쇠** (`duel_rvol_dampen_annual`, 기본 0.50): 기초지수의
연율화 σ20이 문턱 이상이면 사이즈를 절반으로 캡. VIX 위기컷(35)과 **별개의**
리스크 계층 — 위기가 아니어도 섹터 변동성이 높으면 풀레버리지를 막는다.
decide/decide_adaptive 양쪽에 적용, `rvol_damped` 플래그로 카드·원장에 표시.

**필수매수 제약** (`duel_mandatory_pick`, 기본 on): 전 페어가 관망이면 최고확신
종목을 방향성 하프사이즈 콜로 강제 선정(`promote_forced`). 이 시스템의 "관망은
+EV" 기본값을 사용자 정책이 **의도적으로 오버라이드**하는 것이므로 — 정직하게:
① 강제 콜은 `forced=1`로 박제돼 자발 콜과 **분리 채점**되고(`surge forced-eval`
= 제약의 실제 비용 측정), ② 위기(VIX≥35) 관망만은 강제하지 않는다(안전 > 정책 —
3배를 패닉장에 강제 진입시키지 않는다), ③ 강제는 항상 하프사이즈(원치 않은
베팅에 풀레버리지 금지).

## 선행 변동성 레짐 센서 (2026-07-22) — 변동성을 *곡선*으로 읽는 다른 방법

기존 엔진은 변동성을 **하나의 숫자**(VIX 레벨, 기초 σ20)로만 읽었다. 둘 다
*지나간* 변동성을 재는 후행 지표라, 레짐이 **돌아서는 순간**을 놓친다. 사용자
요청("변동성에 영향을 미치는 더 많은 변수를 실시간 파악하는 다른 방법")에 맞춰,
변동성을 **레벨이 아닌 곡선·표면·변화율**로 읽는 센서(`duel/volstate.py`)를 추가:

- **VIX 기간구조** (`^VIX9D`/`^VIX`/`^VIX3M`, 무키): 백워데이션(근월>원월)은 시장이
  *임박한* 스트레스를 원월보다 높게 값매김하는 상태 — 실현 변동성 확대의 대표적
  선행 신호. 콘탱고(근월<원월)가 평온장 기본값.
- **CBOE SKEW** (`^SKEW`, 무키): 좌측 꼬리(급락) 헤지 수요. ATM VIX 레벨이 못 보는
  꼬리 위험의 가격.
- **실현 변동성 가속** (σ5/σ20): >1이면 변동성이 *지금* 20일 기준선보다 빠르게
  팽창 중. 후행 σ20 감쇠가 못 보는 변화율.
- **Garman-Klass 레인지 변동성**: OHLC 전체 범위를 쓰는 실현변동성 추정(종가-종가
  대비 5~8배 효율). 이미 가진 봉에서 계산, 아카이브.

네 신호를 `vol_state ∈ [0,1]`(0 평온…1 스트레스)로 합성. **사이징에만** 작용하는
**선행 감쇠**(`duel_volstate_dampen`, 기본 0.60): 임계 이상이면 후행 σ20이 따라잡기
*전에* 사이즈를 절반으로 캡. 확신을 올리거나 방향을 뒤집는 일은 절대 없어(과적합할
방향 엣지 자체가 없음) `rvol_damped`+`vol_state`로 카드·`선행 변동성 레짐 감쇠`
근거줄에 표시. 데이터 없는 밤엔 하위신호 수만 줄 뿐 오경보 없음(credit/dollar와
동일한 degrade-safe 규율).

**방향성 가설은 별도로 심판.** 백워데이션→약세, SKEW 급등→약세 두 가설은 섀도
`CANDIDATE_FACTORS`(`vix_backwardation`/`skew_tail`)로 등록돼, 다른 모든 후보와
**같은 증거 게이트**를 통과해야만 승격 후보가 된다 — 오늘 밤 라이브 콜은 건드리지
않는다. 캡처된 벡터(vix9d/vix3m/skew/σ5/GK)는 `duel_live_context`에 point-in-time
박제돼 미래 학습기의 원재료가 된다. 나이틀리 학습로그의 `volstate` 섹션에 요약.

## 투자행동 리스크 규율 자가진단 (2026-07-22) — 개인화 사이징 감쇠

"생애설계 7대 축" 아이디어를, 이 엔진이 실제로 엣지를 가진 **재무/투자 행동** 한
축으로 좁혀 리스크 계층에 직결한 것(`duel/discipline.py`). 범용 자기계발 설문이
아니라, 레버리지 트레이더의 실증적 위험행동 6문항(과다레버리지·추격·손절규율·
보복매매·MDD감내·삶-비중)을 자가진단해:

- **항목1–5 → `discipline_factor ∈ [floor, 1.0]`**(`duel_discipline_floor`, 0.40):
  추천 사이즈를 **줄이기만** 한다(≤1.0, 확신·방향 불변 — volstate와 같은 안전 계약).
- **항목6(삶-비중) → 절대 상한**: 하룻밤 노셔널이 사용자가 밝힌 총 레버리지 예산을
  넘지 못하게 캡(② 삶-비중 가드레일).

**순수성 유지**: `decide()`는 DB를 읽지 않는다. 라이브 계층(`live.py`)이
`discipline.active_factor()`/`active_ceiling()`을 읽어 `size_scale`/`size_ceiling`
인자로 주입 — 자가진단이 없으면 1.0/None이라 **옵트인 전까지 콜은 완전히 불변**.

**설문지가 아니라 엔진인 이유**: 각 진단은 `source`('self'/'behavioral')와
타임스탬프로 저장되고 `active_factor`는 **항상 최신 행**을 읽는다. 규율 궤적
(`factor` 시계열)이 "성장추적"을 *채점 가능한 리스크 규율 궤적*으로 재정의 —
대시보드/나이틀리 로그의 `discipline` 섹션. `surge discipline [--assess a,b,c,d,e
--equity X]`.

**행동 앵커링 (phase 2, 2026-07-23 구현)**: 기존 체결원장(`fills`/`orders`)은
duel 콜과 연결이 없고 정체 상태 → 대신 사용자가 콜별 실제 실행 사이즈를 명시
기록(`surge discipline --log DATE PAIR ACTUAL_PCT`). 콜의 권장(size_factor×
duel_size_pct) 대비 실제 비중 = 준수도를 `discipline_adherence`에 적립. 표본이
`duel_discipline_min_n`(20) 이상 쌓이면 **평균 준수도로 행동 factor(≈1/평균준수도,
floor 클램프)를 산출해 'behavioral' 행으로 기록** — 자가보고가 "규율적"이라 해도
실제로 과대베팅(준수도>1)하면 관측된 행동이 진술을 덮어쓴다(OOS 앵커링과 동형).
과소베팅은 벌하지 않는다. 사용자 실입력만 사용 — 날조 없음, 없으면 자가보고 유지.

## 사각지대 루프 (2026-07-15) — 관망은 진단의 시작점

결함 신고(사용자): "저확신이 예상되면 보수적으로 제출을 삼가는 데서 멈추지 말고,
낮은 적중률의 원인을 분석해 추가 예측 상관 변인을 확보·계산에 편입하라. 이것
역시 지속 진화하는 자기개선 엔진의 일부여야 한다." `duel/blindspot.py`가 이
루프를 기계화한다:

1. **진단**: 라벨이 있는 모든 관망 세션을 보관된 컴포넌트 분해로 원인 분류 —
   신호침묵(SILENT, 읽기 부족)/신호충돌(CONFLICT, 강한 신호들이 상쇄)/신호미약
   (WEAK, 정보 없음)/위기가드(CRISIS). 섀도 변형은 관망일에도 방향을 커밋하므로
   원인별 **would-have 적중률이 추정이 아니라 실측**이다.
2. **변인 확보**: 원인별 표적 조건부 변인이 자기 사각지대 세션에서만 발화해
   팩터 레이스에서 채점된다 — `weak_drift`(미약→드리프트), `conflict_asia_tiebreak`
   (충돌→아시아 선행이 타이브레이커), `silent_gap_follow`(침묵→직전 갭 추종).
   CRISIS는 의도적으로 fill 없음(킬스위치를 변인으로 덮지 않는다).
3. **진화 편입**: `surge daily`가 매일 밤 진단+fill 전진 성적을 learning_log에
   박제(`report.blindspot`), `surge blindspot` CLI로 열람. 승격 경로는 다른 모든
   변인과 동일: 팩터 게이트(Šidák) → 피처 벡터 → 설정 레이스 → BASE.

**첫 실측 (라이브 3주 원장)**: 관망 51건 중 49건이 신호충돌이었고, 그날 섀도의
would-have 적중률은 **65%** — 결함 신고가 데이터로 정확히 확인됐다. 충돌일은
정보가 없는 날이 아니라 **타이브레이커 변인이 없는 날**이었다. 그 자리를 겨냥한
`conflict_asia_tiebreak`가 오늘 밤부터 전진 채점을 시작한다.

### 자가 생성 계층 (같은 날 추가) — 재발 패턴이 곧 새 변인이 된다

정적 fill 3종은 씨앗일 뿐이다. `blindspot.generate_fills()`가 매일 밤
(템플릿 라이브러리 × 관망 원인 버킷)을 **point-in-time 컨텍스트 아카이브**
(`duel_live_context`)로 스크리닝하고, 문턱(n≥12, acc≥0.58)을 넘는 조합을
`model_state`에 **자동 등록**한다 — 반예측 템플릿은 `learn.propose_challengers`와
같은 규칙으로 **뒤집어서** 가설화. 등록된 변인은 `factors.all_factors()`로
정적 후보와 함께 매일 기록·채점되고, 승격은 동일한 Šidák 게이트를 통과해야
한다(스크리닝은 등록 필터일 뿐, 심판이 아니다 — 역방향 포함으로 가족 크기가
2배가 되는 비용은 게이트의 k가 지불).

**첫 자가 생성 (라이브 원장, 2026-07-15)**: 시스템이 사람 개입 없이 2개 변인을

  검토한 모든 단일종목 후보의 실패 패턴(숏 다리 유동성 붕괴)과 정반대
- 신호셋 고아 아님: NVDA는 이미 soxl_soxs·tqqq_sqqq·tecl_tecs 3개 페어의
  AMVF `leader` 종목(`baskets.py`) — asia_lead·리더십 신호가 실제로 연결될
  개연성이 labu_labd(바이오텍, 신호 전이 실패 기록됨)와 다르다.
- 이 페어엔 바스켓을 등록하지 않았다 — NVDA 자신이 기초지수이므로 "자기 바스켓
  대비 리더십"은 순환논리가 된다. `framework_features("nvdl_nvd")`는 안전하게
  빈 결과를 반환(크래시 없음), 나머지 신호셋(아시아 선행·VIX·모멘텀 등)으로만 경쟁.

**정직한 현재 상태**: 코드·레지스트리·테스트는 완성됐으나, 이 실행 환경은
yfinance·pykrx 라이브 fetch가 네트워크(TLS) 계층에서 전면 차단돼 있어(이 문서
작성 시점까지 SOXL조차 조회 실패) `price_history`에 NVDL/NVD/NVDA 실 데이터가
아직 없다. 따라서 **이 페어의 백테스트·확신 원장 수치는 존재하지 않는다** —
날조하지 않고 비워둔다. 네트워크가 되는 환경(예: GitHub Actions 러너)에서
`surge duel-archive` 1회 + `surge adaptive --calibrate --offline --pair nvdl_nvd`
를 실행하면 채워진다.

## 구조적 가설

아시아 반도체 대장주(TSMC·삼성전자·SK하이닉스·도쿄일렉트론)는 **미국 세션이 열리기
전에 그날 거래를 마친다.** 한국 저녁 시점에 이들의 당일 종가는 *확정된* 산업 정보다.
이를 전일 미국 모멘텀/추세, 변동성 레짐(VIX), 금리 임펄스와 함께 투명 가중 투표로
합성한다. (라이브 한정: 나스닥 선물 오버레이.)

## 룩어헤드 차단 — 구조로 강제

- 미국 시장 피처는 전부 `.shift(1)` — D일 행에는 D−1 종가까지의 정보만 존재.
- 아시아 피처만 당일(D) 사용 — 시간대가 누수를 물리적으로 차단(아시아 D 종가는
  미국 D 시초가보다 먼저 확정).
- SOXX 시가→종가 수익률은 **라벨로만** 사용.
- 백테스트는 운영 룰을 *그대로* 리플레이.

## 신호 (가중 투표, 전 구성요소 근거 공개)

| 신호 | 가중치 | 내용 |
|---|---|---|
| asia_lead | 0.35 | 아시아 4사 당일 수익률(변동성 정규화, 시총 가중) |
| trend | 0.15 | SOXX 50일선 이격 |
| momentum_5d | 0.15 | SOXX 5일 모멘텀(z) |
| vix_regime | 0.15 | VIX 수준 + 급변 |
| rates | 0.10 | 미 10년물 일간 변화(역방향) |
| mean_reversion | 0.10 | 전일 ±2σ 초과 시에만 역방향 |
| futures | 0.20 | **라이브 전용** NQ 선물(백테스트 제외 — 보수적) |

부재 구성요소(아시아 휴장 등)는 가중치 재정규화로 제외된다.

## 결정 규칙

- **위기 변동성**: VIX ≥ 35 → 무조건 관망 (3배 상품 + 패닉 = 도박)
- **확신도 밴드**: |score| < 0.15 관망 · 0.15~0.35 절반 사이즈 · ≥ 0.35 풀 사이즈
- **브래킷**: 진입 후 손절 = 1.0×ATR14 아래, 익절 = 1.5×ATR14 위 (R:R 1.5),
  미체결 시 **종가 청산 — 오버나이트 금지**(레버리지 ETF 복리 손실 + 갭 리스크 회피)
- **사이징**: 자본의 최대 10% × 사이즈 팩터 (3배 상품이므로 보수적)
- 양쪽 다리 모두 **롱**(SOXL=3x불, SOXS=3x베어)이므로 브래킷은 항상 롱 방향

## 사용법

```bash
uv run surge duel              # 오늘 밤 판정 (저장됨; 선물 오버레이 포함)
uv run surge duel-backtest --period 2y
uv run surge duel-eval         # 지난 콜 사후 채점 → 누적 적중률
```

매일 스케줄 잡(`daily-surge-snapshot`)이 `duel-eval` → `duel`을 자동 실행한다 —
**모든 콜은 세션 전에 저장되고 세션 후에 채점된다.** 이것이 정직성의 핵심 루프다.

## 2026-06-11 기준 2년 백테스트 — 정직한 결과

452일 중 293일 베팅 / 159일 관망 (현실적 슬리피지 2bp):

- 방향 적중률 **54.3%** (z=+1.46 — 동전던지기와 통계적으로 미구분)
- 수익 **−3.4%**, Sharpe −0.42 · 오라클 한계는 +1739% (상금은 크나 신호가 없음)
- **풀사이즈 적중률 48% < 절반사이즈 58%** — 확신도가 역작동
- IC: vix_regime −0.141, **asia_lead −0.077**, momentum −0.057 …

**핵심 발견**: 아시아 신호의 정보는 실재하지만 **미국 시초가 갭에 선반영**되고,
한국 저녁의 행위자가 실제로 잡을 수 있는 시가→종가 구간은 그 갭을 오히려 약하게
*되돌린다*. 즉 "신호가 없다"가 아니라 **"실행 가능한 시간대에 신호가 남아있지
않다"**가 현재까지의 결론이다.

이 표본에서 IC 부호를 뒤집어 재적합하는 것은 **과최적화이므로 하지 않는다.**
전진 기록(`duel-eval` 누적)이 심판이다. 관망 밴드가 유일하게 확실한 +EV 장치다.

## 갭 원인 분석 (`surge duel-gap`, 2026-06-11 가동)

채점된 모든 콜에 대해 **예측-실측 갭의 원인**을 자동 분해한다 (아침 잡에 상시 편입):

- **갭 선반영**: 신호 방향이 시초가 갭(전일 종가→시가)에 흡수된 뒤 장중 역행 —
  핵심 구조 가설을 콜 단위로 계측
- **휩쏘 스탑아웃**: 방향 적중인데 브래킷 손절로 실행 손실 (신호 문제 아님)
- **주범/견인 신호**: 틀린(맞은) 쪽 기여 최대 컴포넌트 식별
- **저확신 경계선 베팅** / **관망 기회비용** (관망 정당 vs 큰 움직임 놓침)
- **신호별 방향 일치율**: 라이브 전진 기록 누적 — 인샘플 백테스트가 자신을 채점하지
  못하게 하는 장치. 콜 저장 시 컴포넌트가 구조화 JSON으로 함께 박제됨(`components`).

첫 분석(6/10 관망 콜): 실측 −1.9% — futures/momentum/vix는 방향 일치, trend/rates가
오답 측. "큰 움직임 놓침"으로 분류(확신도 미달 차단). n이 쌓이며 이 표가 신호 품질의
실질 심판이 된다.

## 섀도 변형 A/B — 루틴이 정확도를 *개선*하는 메커니즘 (2026-06-13)

루틴은 champion을 측정만 하지 개선하지 않는다(소표본 재조정=과적합). 개선의
유일한 안전 경로는 **전진 A/B 테스트**다: 매일 밤 champion과 함께 여러 변형이
*동일 컴포넌트*를 재집계해(추가 페치 0) 방향을 커밋하고, duel-eval이 실측 라벨로
함께 채점한다. 변형 7종 × 페어 4 = 매일 28개 데이터포인트가 누적돼 champion-only
대비 ~28배 빠르게 통계적 판정에 도달한다. (`src/surge/duel/variants.py`)

변형은 진단된 약점을 직접 공격한다:
| 변형 | 가설 |
|---|---|
| `drop_momentum` | momentum_5d(전진 일치율 12%)는 잡음 → 제거 |
| `inv_momentum` | momentum이 역신호인가? → 부호 반전 |
| `vix_futures` | 최강 2신호(vix·futures)만 |
| `asia_x2` | 구조적 아시아 선행 강화 |
| `no_trend` / `defensive` | trend 제거 / 약신호 제거+vix 강화 |

`surge variants`로 리더보드 확인. **승격은 자동 아님** — n≥30 독립 채점일 +
2비율 z≥1.64(95%)를 넘은 변형만 추천되고, `surge variants --promote <name>`로
사람이 적용하면 그 시점부터 champion이 새 가중치로 전진 채점된다(`model_state`).
`--reset`으로 복귀. 콜은 항상 방향 커밋(신호 스킬 비교용)이며, champion 실거래
정확도는 관망 포함 별도(duel-eval).

> 핵심: 이제 **매 루틴이 7개 대안 가설을 누적 검증**한다 — 측정만 하던 루프가
> 과적합 없이 개선을 *생성*하는 루프로 바뀌었다.

## 적응 엔진 업그레이드 (2026-07-02) — 워크포워드 학습 가중치

고정 가중치 챔피언의 진단된 3대 결함을 정면 공격한 업그레이드:

1. **가중치가 손으로 고정** — IC가 음(vix_regime −0.14 등)인 신호도 그대로 투표.
   인샘플 부호 뒤집기는 과최적화라 금지였는데, 그 딜레마의 정직한 출구가
   **워크포워드 릿지 학습**(`duel/adaptive.py`): 매 세션 D의 예측은 *D 이전
   세션만으로* 학습한 모델에서 나온다(expanding window, min_train=120,
   closed-form ridge + Platt 확률 보정, sklearn 무의존). 부호가 뒤집히더라도
   그 시점까지의 데이터가 시킨 것 — **전 구간 아웃오브샘플**이다.
2. **확신도 역작동**(풀사이즈 48% < 절반 58%) — 점수가 확률이 아니었기 때문.
   적응 엔진의 확신도는 보정된 |2p−1|: 밴드가 의미를 되찾는다(아래 결과).
3. **라벨-피처 불일치** — 라벨은 시가→종가인데 피처는 종가→종가(오버나이트 갭
   혼입). 인트라데이 분해 피처 3종(`f_oc_mom5`·`f_oc1`·`f_gap1`, 전부 D−1
   시프트)을 학습기에 공급.

### 검증 결과 (price_history 아카이브 오프라인 리플레이, 슬리피지 2bp)

| 구간/페어 | static(고정 가중치) | adaptive(전진 OOS) |
|---|---|---|
| SOXL/SOXS 전체(2010~) | 51.1% · −11.1% · Sh −0.15 | **52.9% (z=+2.53) · +28.9% · Sh 0.57** |
| SOXL/SOXS 2019~ 신규시작 | 51.2% · −5.3% | **53.4% (z=+2.26) · +40.6% · Sh 1.17** |
| TQQQ/SQQQ 전체 | 51.9% · −1.0% | **53.6% (z=+3.40) · +5.0%** |
| TECL/TECS 전체 | 52.1% · −8.8% | **54.1% (z=+3.64) · +0.8%** |

- 확신도 밴드 복원: 풀사이즈 62.7% > 절반 52.6% (전체 구간; 2019+에서도 57.5% > 51.6%)
- 정직한 해석: 여전히 53~54%의 **얇은 엣지**다. 다만 4페어·복수 구간에서
  일관되게 z>2로 재현되고, PnL 부호가 뒤집혔으며, 사이징이 처음으로 정확도와
  같은 방향을 본다. "완벽한 예측"은 존재하지 않는다 — 이 표가 상한이 아니라
  현재 위치다. 최종 심판은 언제나 전진 기록.

### 갭 가드 — 가설 기각 기록 (기본 OFF)

"신호가 시초가 갭에 선반영되면 진입 취소" 가설을 기계화(`--gap-guard Z`,
콜에 조건 박제 + duel-eval이 기계적으로 판정)해 전체 아카이브에 리플레이한
결과: **차단됐을 트레이드가 오히려 59% 적중, would-have PnL 크게 양수** —
방향성 갭은 되돌기보다 지속한다. 가설은 기각됐고 기본값은 0(off)이다.
부호를 뒤집어 "갭 확인 필터"로 쓰고 싶은 유혹이 곧 인샘플 과적합이므로
하지 않는다. 기계는 남겨서 누구든 재측정할 수 있다.

### 재귀적 자기개선 계층 (2026-07-02 확장) — "변인 추정"의 기계화

전제: 완벽한 예측은 없다. 대신 **예측→적중 분석을 십수년 아카이브와 매일 밤의
현재 데이터로 반복**하면 변인(어떤 정보가 익일 방향을 움직이는가)의 추정이
누적되고, 그 추정 과정 자체가 기록·검증·개선된다. 이를 세 겹으로 기계화:

1. **추정기 자체가 가설** (`adaptive.CONFIGS`): 기억 길이(expanding vs 2y/4y
   롤링), 수축 강도(λ 1/4/16), 변인 가족(전체/투표만/인트라데이만) 7종 설정이
   매일 밤 *각자* 방향을 섀도 커밋하고 동일 전진 A/B에서 채점된다. "모델이
   어떻게 배워야 하는가"를 논쟁이 아니라 누적 증거가 결정.
2. **변인 가중치 박제** (`adaptive_weights` 테이블): 기본 설정 학습기가 매일
   밤 적합한 피처별 가중치를 point-in-time 저장. `surge adaptive`가 현재
   credit되는 변인·드리프트·부호반전을 보여주고, `surge daily`의
   learning_log에 요약이 남는다 — 변인 추정의 고고학적 기록.
3. **변인의 독립 심판** (shadow factors): 인트라데이 분해 변인 3종
   (intraday_mom·prev_gap_follow·prev_intraday_follow)이 학습기 내부뿐
   아니라 단독 팩터 레이스에도 등록 — 학습기가 credit하는 변인이 단독으로도
   전진 예측력이 있는지 교차 검증.

**16년 리플레이 설정 레이스** (전량 OOS, 항상-커밋, `surge adaptive --replay --offline`):

| 설정 | SOXL/SOXS | TQQQ/SQQQ | TECL/TECS |
|---|---|---|---|
| adaptive(기본) | 51.3% (z=+2.0) | **53.2% (z=+5.2)** | 51.9% (z=+3.2) |
| adaptive_roll4y | **52.0% (z=+3.1)** | **53.2% (z=+5.2)** | 51.9% (z=+3.1) |
| adaptive_intraday | 50.7% | 52.9% | **53.2% (z=+5.3)** |
| adaptive_roll2y | 51.4% | 51.2% | 51.6% |

정직한 판독: **보편 승자는 없다** — 반도체는 4년 기억, 나스닥은 긴 기억,
기술셀렉트는 인트라데이 변인이 우세하고, 2년 기억(roll2y)만 일관되게 열세다.
리플레이가 후보를 좁히고, 최종 결정은 밤마다 쌓이는 전진 기록이 내린다 —
이것이 과적합 없이 개선이 *생성*되는 재귀 구조다.

### 확신 구간 원장 + OOS 앵커링 (2026-07-02) — 확신을 관측 적중률에 정박

"P(상승) 56%"라는 주장은 그 구간의 콜이 실제로 몇 %나 맞았는지와 함께가
아니면 무의미하다 — 사용자가 직접 계산하게 두는 대신 시스템이 계산·박제·인용:

- **원장** (`adaptive_calibration`): 확신 구간(50–53/53–55/55–58/58–62/62+)별
  적중률을 리플레이(16y, 전량 OOS)와 전진 기록 양쪽에서 유지.
  `surge adaptive --calibrate --offline`가 갱신(파이프라인 야간 편입).
- **발견**: 최초 실측에서 보정 곡선이 단조가 아니었다 — "55–58%" 주장 구간이
  실제 49.4%(동전 이하), "62%+"가 51.5%. 학습창 Platt는 자신의 OOS 성적을
  모른다.
- **처방 — OOS 앵커링** (`adaptive.recalibrate_prob`): tonight의 원시 확률을
  *과거 아웃오브샘플 예측들*의 해당 구간 관측 적중률로 재사상(스무딩 m=100,
  방향 보존 플로어 0.5+ε). 원장 자체가 보정기가 되는 재귀 —
  워크포워드 내부에서도 과거 OOS만 사용하므로 누수 없음.
- **결과** (SOXL/SOXS): 주장-실적 괴리 ±7%p → **1%p 미만**. 앵커링 후
  53–55% 주장 구간 실측 53.8%(n=320), 55–58% 구간 54.8%(n=378).
  58%+ 주장은 소멸 — **시스템이 입증 못 하는 확신을 더 이상 말하지 않는다**.
  LABU는 전 구간이 바닥에 깔림(콜 거부) — 연구 전용 판정과 일치.
- **밴드 재정의**: 앵커링된 척도에서 |2p−1|은 곧 입증된 엣지다 —
  0.05(=관측 52.5%+) 절반, 0.10(=관측 55%+) 풀. 밴드 적용 시 6,101세션 중
  660일만 베팅(선택과 집중), 적중률 52.9%, +10.8%, 풀사이즈 구간 54.2%.
- **카드 인용**: 매일 밤 카드가 "적응 엔진: P(상승) 55.1% — 이 확신
  구간(55–58%)의 과거 적중률: 54.8% (리플레이 n=378) · 전진 n=…"을 명시.
- **정보 확장(무키)**: FOMC 결정일 2015–2027(연준 공식, 사전 확정) +
  pre-FOMC 드리프트 피처, QQQ 상대강도(20d), 요일 변인. 라이브 전용 읽기
  (NQ선물 등)는 `duel_live_context`에 point-in-time 박제 — 몇 달 쌓이면
  학습 가능한 변인으로 전환된다.
- **기술 계층 + 옵션 흐름(무키, 2차 확장)**: RSI(14)·상대거래량(RVOL20)을
  D−1 시프트로 계산해 피처 벡터(16)에 편입하되, **기본 엔진에는 넣지 않았다**
  — 16피처 리플레이가 55–58% 구간을 다시 비단조(48.2%)로 만들었기 때문.
  기본은 검증된 14피처(BASE_FEATURES)를 유지하고, 기술 계층은
  `adaptive_tech` 설정으로 전진 레이스에서 자리를 증명해야 한다.
  `rsi_reversal`은 팩터 레이스에 별도 등록. 옵션 체인 스냅샷(ATM IV·풋/콜
  OI·거래량 비율, yfinance 무키)은 `options_snapshots`에 매일 밤 박제 —
  히스토리가 없는 소스라 전방 축적만 가능하며, 몇 달 뒤 후보 변인으로
  승격된다. 리더(NVDA 등) 실적일 근접도도 라이브 컨텍스트에 함께 박제.
  **(2026-07-22 수리)** 아카이브가 07-03 이후 정체 → Yahoo `/v7/finance/options`가
  crumb+쿠키 인증을 요구하도록 바뀐 게 원인(quotes.py가 쓰는 chart 엔드포인트는
  crumb 불필요라 조용히 살아 있었음). `options._yahoo_crumb`가 A1/A3 쿠키를
  프라임하고 `/v1/test/getcrumb`로 crumb을 받아 붙인다 — 나이틀리 러너에서 검증.

### 사용법 + 승격 경로

```bash
uv run surge duel-backtest --offline --compare        # 2×2 verdict 테이블
uv run surge duel-backtest --offline --adaptive       # 적응 엔진 단독
uv run surge duel-backtest --offline --config adaptive_roll4y  # 특정 설정
uv run surge duel-backtest --offline --gap-guard 1.0  # 가드 재측정
uv run surge adaptive                                 # 변인 가중치 + 드리프트 + 레이스
uv run surge adaptive --replay --offline              # 16y OOS 설정 레이스
```

- 매일 밤 `surge duel`이 적응 엔진의 콜을 **섀도 변형 `adaptive`**로
  `duel_variants`에 자동 기록 — 기존 A/B 레이스에 합류해 전진 채점된다.
- 프로덕션 전환은 자동이 아니다: 전진 기록이 게이트(n≥30, Šidák z)를 넘고
  **사람이 `SURGE_DUEL_USE_ADAPTIVE=1`을 설정**해야 적응 엔진이 실제 콜을
  낸다(`model` 컬럼에 엔진 박제). `variants --promote adaptive`는 의도적으로
  거부된다 — 이 스위치는 환경변수 하나로 명시적으로.
