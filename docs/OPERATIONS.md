> **정보·도구 제공 — 투자자문 아님 · 수익 보장 없음.** 모든 예측은 누수 없이

> 산출물은 추천이 아니라 가설이다.**

## 0. 먼저 이것부터 — 검증 게이트 (`surge verdict`)

다른 모든 명령보다 먼저 본다. **"이 시스템을 믿고 돈을 걸어도 되는가?"** 한 줄로 답한다.

```
surge verdict
```

각 전략을 **순진한 기준선**과 비교해 운(luck)으로 설명되지 않는 초과가 있는지만 본다:

| 판정 | 뜻 | 행동 |
|---|---|---|
| ⭐ **신호** | 🟢 + 5조건(표본 n≥30·기간분산 4주+·**증거 e≥20(anytime-valid)**·**전·후반 모두 유지**·비용차감 순익>0) 모두 충족 | **비로소 베팅 참조 가능** |
| 🟢 엣지 | 기준선을 통계적으로(z≥1.64) 초과 — 단 ⭐ 조건 미충족 | 통계적 엣지지만 **아직 가설** |
| 🟡 미검증/엣지없음 | 기준선 이하거나 유의성 부족 | 가설 — 베팅 근거 아님 |
| ⛔ 음의 엣지 | 기준선보다 **유의하게 나쁨** | 그 전략 신호를 따르지 말 것 |
| 🔴 데이터 부족 | n<30 | 판단 보류, 표본 축적 대기 |

> **🟢는 "통계적으로 기준선과 구별됨"일 뿐 신호가 아니다.** 돈을 거는 *신호*는 ⭐ —
> 한 방향 30일은 동전도 🟢를 통과하므로, 표본·레짐·비용·비감쇠·증거(e값)를 모두
> 통과해야 비로소 참조한다. 현재 ⭐는 0개(아무 전략도 격상 전).

> **신호 검증을 정직하게 앞당기는 법 — anytime-valid e-값 (peek-safe).** 고정 n의
> z-검정은 *미리 정한 n에서만* 유효하다. 매일 들여다보며 "통과하는 첫날" 격상하면
> 거짓양성률이 몰래 부풀어 오른다(optional-stopping 편향 — 백테스트가 거짓말하는 고전적
> 경로). **e-값**(plug-in 검정 마팅게일)은 그 결함이 없다: Ville 부등식으로
> `P(sup_n e_n ≥ 1/α) ≤ α` 이므로 **e ≥ 20(=1/0.05)일 때 격상**하면 *어떤 멈춤 규칙에서도*
> (= 매일 점검) 5% 유의수준이 보장된다. 그래서 **진짜 엣지는 증거가 결정적이 되는 즉시**
> 정직하게 격상되고, 노이즈는 임계를 못 넘는다. 단, anytime-valid는 *엿본* 검정보다 표본이
> ~2배 더 든다 — 이는 자기기만을 피하는 비용이지 결함이 아니다(거짓 없음). 실제 달력시간
> 가속은 (a)이 e-값 + (b)기간분산 4주 하한 + (c)**독립 시행 수**(다변화된 6개 페어·KR·다시장)에서
> 나온다. 대시보드 ‘신호증거 미터’가 이 e값 진척을 0→100%(e=1→20)로 시각화한다.

> **증거: 게이트는 이항 e-값 하나, 연속·페어는 보조진단(모두 peek-safe).** ① **이항**
> (`evalue_bernoulli`): 방향 적중. **이 트랙 하나가 ⭐ 증거 게이트를 판정한다.** ② **연속**
> (`evalue_returns`): 수익 *크기*까지 쓰는 hedged-capital 베팅 마팅게일 — H0 평균≤0 vs
> H1 평균>0, *상단만* 윈저라이즈(하단은 절대 끌어올리지 않음 — 그러면 평균이 올라가 검정이
> 깨짐)로 surge의 +500% 로또가 가짜 엣지를 못 만들게 함(보수적). ③ **페어**(`paired_evalue`,
> McNemar): 같은 세션에서 전략 대 ‘항상 강세’ 직접 대조 — *추정* 기준선 오차를 제거. ②③는
> **게이트에 넣지 않고 보조진단으로만 표시**한다. 이유(정직): FWER(전체 1종오류) 통제하에선
> **공짜가 없다** — 트랙을 게이트에 더하면 이항의 검정력을 *깎을 뿐*이다. (앞서 ‘세 트랙
> 평균’으로 게이트한 시도는 강한 이항 엣지를 약한 트랙이 임계 아래로 *희석*해 ⭐를 더
> 어렵게 만들었다 — 목표와 반대.) 순진한 ‘이항≥20 OR 연속≥40’도 무효다(전체-α 임계와 보정
> 임계를 섞으면 FWER가 α를 넘음). 유효한 Bonferroni ‘OR’는 모든 트랙을 k/α(2트랙이면 각
> 40)로 올려 이항 막대를 되레 높인다. 그래서 **게이트는 보정된 단일 이항 검정으로 유지**하고
> ②③는 *확증 정보*로만 쓴다. 모두 한쪽 검정이라 기준선 이하 전략은 **증거 0**(거짓 신호 불가).

**기준선이 핵심이다 (소매용 백테스트가 거짓말하는 이유).**
- **duel** → "항상 강세 레그 매수"(반도체 우상향). 동전(50%)이 아니라 *이것*이 기준.
- **rotation** → 게이트를 **통과 못 한** 후보의 T+5 도달률. 필터가 뭔가 더하는가?
- **surge** → 후보 익일 **상승 비율 vs 0.5 동전**. ("+100% 도달률 vs 시장 기저율"은
  함정 — 마이크로캡을 골랐다는 것만 증명할 뿐, 평균은 소수 대박이 부풀린다.)

### 1.1 매일 자기개선 루프 — `surge daily` (프로그램이 스스로 개선하는 심장박동)
**하루 1회** 닫힌 루프를 돈다: ① 어제 콜 **채점**(증분 — 새로 성숙한 결과만, 2년 재백필 아님) →
② **진화**(`learn.register_discovered`가 자기 진단에서 새 가설을 발굴해 전진 검증에 등록 — 되돌릴 수
있고 라이브 영향 없음) → ③ **판정**(`verdict.assess` 증거 재계산) → ④ **기록**(`learning_log` 1행 +
전일 대비 변화). 대시보드 **🧠 매일 자기개선 로그** 패널과 `/api/learning`이 이 타임라인을 보여준다.
- **안전 경계(비협상)**: 루프는 채점·진화·판정·기록만 자동으로 한다. **전략의 라이브 승격·설정
  변경·주문·송금은 절대 자동이 아니다** — `learn.gate` + 승인 큐로 **사람이 승인**한다. 리포트는
  승격 후보를 *표시*만 한다(`promote_ready`).
- **수동 1회**: `\.venv\Scripts\surge.exe daily`  ·  **예약(OS 작업 스케줄러, 매일 07:55 KST)**:
  프로그램 `\.venv\Scripts\surge.exe`, 인수 `daily`, 트리거 매일 07:55, `StartWhenAvailable` 체크.
- 런타임은 채점 단계의 시세 조회에 비례(증분이라 보통 수 분; 장기 미실행 후 백로그면 길어짐).
  degrade-safe — 한 단계가 실패해도 한국어 경고 후 계속, 프로세스는 죽지 않는다.

## 3. 결과 리포트 — 읽는 법

| 명령 | 보여주는 것 |
|---|---|
| **`surge verdict`** | **먼저 본다** — 전략별 기준선 초과 판정(🟢/🟡/⛔/🔴) |
| `surge report` | **원스크린** — 오늘 duel 콜·누적 적중·최근 채점·신호 일치율·갭·변형 A/B + 게이트 헤드라인 |
| `surge duel-eval` | 페어별 누적 적중률·평균손익 |
| `surge duel-gap` | 콜별 오답 원인(갭선반영/휩쏘/주범신호) + 신호 방향 일치율 |
| `surge variants` | duel 가중치 변형 리더보드 + 승격 후보 |
| `surge factors [--backfill]` | **섀도 팩터** 리더보드 — *어떤 미사용 신호를 추가할까* (백필=2년치 즉시) |
| `surge rotation --why` | 한국 회전 후보(통과/근거), "핫노드 제외·후방노드 우선" |
| `surge rotation-eval` | T+1/T+3/T+5 누적 + 변형 A/B(top-3 픽 평균) |
| `surge kr-factors [--backfill]` | **KR 어텐션 섀도 팩터** — 네이버 검색급등이 +10%/T+5를 선별하나(DataLab 백필) |
| `surge krx-check` | 한국 데이터 연결·자격증명 진단 |
| `surge watch --horizon short` | 큐레이션 타깃 **단기(≤1주)** 매수존/손절/목표(ATR) |
| `surge watch --horizon swing` | **스윙(1주~3개월)** MA추세+채널 가격대 |
| `surge watch --horizon long` | **장기 멀티배거** 구조적 도시어(시총·낙폭·테마·10x여지, *확률 없음*) |

> `watch`는 **추적 계층** — 소수 타깃의 기계적 가격대/도시어만 제공하며 예측·학습
> 루프(duel/rotation)를 건드리지 않는다. 가격은 ATR/추세 기반 산출(추천 아님),
> 장기는 확률 부여 없는 모니터링용. 타깃 편집: `src/surge/watch/targets.py`.

## 4. 학습 루프 (정확도가 *개선*되고 프로그램이 *진화*하는 메커니즘)

측정(매일 채점)과 **개선**(섀도 변형 A/B)이 분리돼 있고, 승격 판정은 단일
정직한 게이트(`src/surge/learn.py`)가 담당한다 — duel·rotation 공통.

**(a) 전진 A/B 측정** — 매일 champion과 함께 여러 가중치 변형이 같은 데이터를
재집계해 전진 채점된다(추가 페치 0). 누수 없음(컴포넌트는 세션 전 동결).

**(b) 정직한 승격 게이트 `learn.gate()`** — 도전자는 다음을 *모두* 충족해야만
승격 후보가 된다. 하나라도 못 넘기면 표시되지 않는다:
1. **champion 초과** (2-비율 z) — 단, **Šidák 다중검정 보정**: 변형 k개를
   경쟁시키면 요구 z가 1.64 → (k=7일 때) ≈2.43으로 상향. *보정 없는 7개 동시검정은
   모두 동일해도 약 30% 확률로 거짓 승격을 낸다 — 이 구버그를 막는다.*
2. **기준선 초과** (always-bull/bear, 또는 rotation은 풀 전체 도달률). champion만
   이기는 것은 "지는 신호의 최선 과적합"일 수 있다(verdict 게이트가 잡아낸 함정).
3. **n≥30** 독립 채점.
- 승격은 **자동이 아님** — 게이트 통과 후보만 리포트에 표시, 사람이 `--promote` 적용.
- **레짐 점검**: 최근 n건 롤링 정확도를 함께 표시해 전체평균이 숨기는 성능 저하를 노출.

**(c-2) 섀도 FACTOR — "어떤 팩터를 *추가*할까" (`duel/factors.py`, `surge factors`).**
변형(c)은 *있는* 팩터를 재가중할 뿐이다. 섀도 팩터는 라이브 투표에 **없는 후보 신호**
(asia_breadth·und_accel·pullback·vix_meanrev + 크로스에셋 credit/dollar/bonds)를 매
세션 기록·전진 단독 채점한다(라이브 결정 미관여·누수 0). 후보는 **자기 세션의 always-bull
기준선을 Šidák 보정 유의수준으로 초과**할 때만 사람이 `signals.WEIGHTS`에 추가 — 노이즈
팩터 자동 차단. `surge factors --backfill`이 2년치(451세션)를 즉시 채워 *수주 대기 없이*
"이 팩터를 추가했어야 하나"에 데이터로 답한다(현재 어떤 후보도 기준선 미초과 → 추가 없음).

**(c) 자기-진화 `learn.register_discovered()`** — 가설 집합이 더 이상 고정 7종이
아니다. 매일 채점 후 시스템이 **자신의 전진 진단**(컴포넌트별 라이브 부호 일치율)을
스캔해 반(反)예측적 컴포넌트를 자동으로 가설로 등록한다 — *강도에 맞춰*: 강하게
역상관(≤0.40, 예: momentum_5d 12%)이면 `disc_inv_<c>`(부호 반전), 동전 근처
(0.40~0.45)면 `disc_drop_<c>`(가중치 0). 기존 정적 변형과 동일한 가설은 중복
등록하지 않는다(Šidák k 불필요 상향 방지). 이 가설은 즉시 전진 경쟁에 투입되며
(🔬 표시), 승격은 동일 게이트로 사람이 결정한다 — 코드를 손대지 않아도 새 질문이 생성된다.

## 5. 사용자 투자 참조 절차

각 단계는 **CLI** 또는 **대시보드 패널**(§3.1) 중 편한 쪽으로 본다 — 같은 데이터다.

| 시점 | 보는 것 (CLI · 대시보드 패널) | 행동 기준 |
|---|---|---|
| **매번 먼저** | `surge verdict` · **검증 게이트(맨 위)** | 🟢가 하나도 없으면 *그 어떤 산출물도 참조용 정보일 뿐* |
| 미국: 21:50 KST | `surge report` · **오늘밤 duel 콜** | **풀사이즈 + 근거 정렬**일 때만 의미. 관망=정상 출력. labu_labd는 연구전용 |
| 한국: 16:05 후 | `surge rotation --why` · **rotation 후보** | 통과(✓)만 1순위 관찰. 부분충족=연구. **이미 급등한 핫노드 추격 금지** |
| 다음날 아침 | `surge duel-gap` (CLI 전용) | 행동 없음 — "왜"만 축적 |
| 주 1회 | `surge variants`·신호 일치율 (CLI 전용) + **n** | n<30이면 결론 금지 |
| 월 1회 | `surge verdict` 판정 변화 · **검증 게이트** | 🟢로 바뀐 게 있는가? **없으면 베팅하지 않는다** |

**불변 규율 4가지**
1. **일일 결과로 판단하지 않는다** — 일일은 읽기용, 결정은 월간 누적.
2. **적중률보다 기준선 초과 여부** — 강세장은 동전도 60% 맞는다. `surge verdict`의 '초과(%p)'만 본다.
3. **소표본 재튜닝 금지** — 변형 승격 게이트(n≥30)를 신뢰.
4. **검증 전 산출물은 추천이 아니라 가설** — **⭐(신호) 전까지** 베팅 근거 없음. 🟢(통계적 엣지)도 5조건을 통과해 ⭐가 되기 전엔 가설이다.

## 6. 정직한 현재 상태 (2026-06-14, `surge verdict` 실측)

| 전략 | n | 판정 | 의미 |
|---|---|---|---|
| duel (US 레버리지) | 5 | 🔴 데이터 부족 | 표본 부족 — 결론 불가 (~6월 말 충분) |
| rotation (KR 회전) | 0 | 🔴 데이터 부족 | 방금 가동 — 첫 T+5 신호 ~6/22 |
| surge (US +100% 점화) | 588 | ⛔ 음의 엣지 | 후보 익일 상승률 **45% < 50% 동전** |

- **핵심 진실:** 이 프레임들은 *올바른 모집단/노드를 격리*하지만, **검증된 예측 엣지는
  아직 없다.** surge는 충분한 표본(588)으로 *동전보다 나쁨*이 입증됐다 — +100% 대박은
  팻테일에 존재하나 *전형적 후보는 손실*이다. duel/rotation은 아직 판단 불가.
- 시스템의 진짜 가치는 **그 사실을 돈을 잃지 않고 정직하게 측정·게이트**하는 데 있다.
  `surge verdict`가 🟢를 내놓기 전까지, 모든 출력은 *정보이지 신호가 아니다.*
